可做研究点

复现并比较“隐式程序性记忆”与“显式文本记忆”在同一 agent 任务中的失效模式，而不只比平均分数。

NPM 最有价值的主张不是简单提分，而是缓解 text-action disconnect。若只看 benchmark 均分，很难知道隐式 steering 是否真的改善了执行层面的决策一致性、长程依赖或工具调用稳定性。

第一步：选一个公开 agent benchmark 和一个强文本记忆/RAG 基线，先不实现完整 NPM，只做最小版对照：把历史成功/失败轨迹分别压成简单的 steering 信号或检索文本，统计两类方法在同一任务上的错误类型分布，如计划错误、工具选择错误、重复试错、上下文遗忘。

风险：若原论文使用的 steering 提取细节未公开，最小复现可能无法达到其效果；同时错误类型标注需要额外人工或规则设计。

做一个“路由安全压力测试”基准，专门比较基于 profile 的 router 与基于行为探测的 router 在恶意 agent 场景下的鲁棒性。

Linguistic Firewall 的关键价值在于安全，不仅是常规任务分配精度。当前很多 MAS 工作默认 agent 自述可信，但真实系统中最脆弱的正是这一点。

第一步：构造一个小规模多 agent 沙盒：让部分 agent 通过提示词伪装成高能力、或在特定任务上植入后门触发条件；分别用文本画像路由和简单能力测评路由做任务分配，比较成功率、误路由率和被攻击后的性能下降。

风险：行为测评路由本身有成本，探测题设计不当时可能只测到表面能力；此外恶意行为的模拟是否足够真实会影响结论外推。

把 SrDetection 的“原样本-语义等价变体”思想迁移到自然语言推理或数学 benchmark，检查数据泄漏检测是否具有跨模态普适性。

如果自参照泄漏检测只在代码任务有效，其方法学影响有限；若可扩展到一般语言任务，就能成为更通用的 benchmark 清洗与模型审计工具。

第一步：选择一个带标准答案的 NLP benchmark，为每个样本生成少量高质量语义等价改写，先在开源模型上比较原题与改写题的正确率差、置信度差或一致性差，观察这些差异是否能分离已知可能记忆的样本。

风险：自然语言任务的“语义等价改写”比代码更难严格控制，改写可能不小心改变难度；没有已知泄漏标签时也难评估检测准确率。

在动作-语言连续学习中增加“任务边界不明确”的评估，检验无任务标签路由是否真的接近真实部署。

连续学习论文常在清晰任务切分下报告低遗忘，但真实输入往往跨语义簇、边界模糊。若路由器只在理想分段数据上有效，其实际价值会被高估。

这篇论文针对将大语言模型从“静态求解器”转变为可持续与环境交互的自主智能体这一任务，提出了Neural Procedural Memory（NPM）框架，用历史对比经验提炼出的激活空间steering vectors来充当程序性记忆，而不是仅靠RAG式显式文本指令。根据摘要，作者在4个agent benchmark上评估了这种隐式激活引导，并指出它与显式指令基线表现相当，且两者结合可能更优。

Photopic vision, including fast motion and color perception in daylight, is mediated by cone opsins, specialized G protein–coupled receptors (GPCRs).
